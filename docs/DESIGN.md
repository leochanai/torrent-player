# DESIGN.md

## 设计来源与参考 DNA

- 设计来源：用户确认的 UI 彻底重设计方向 + 项目产品边界。
- 参考来源：
  - 主视觉 DNA：黑匣子信号甲板，用于表达 WebTorrent 来源注入、媒体监视器、peers/Service Worker 信号和错误诊断。
  - 产品行为参考：WebTorrent Desktop，用于 magnet / `.torrent` 添加、边下边播、文件列表和 peer 网络心智。
  - 播放器控件参考：Mux Player for web，用于响应式播放器控件、按播放器尺寸隐藏不适合控件、fullscreen / picture-in-picture / volume 等基础行为。
- 参考 DNA 摘要：
  - 设计系统：碳黑底色、骨白文字、酸性黄绿主操作、冷青信号、琥珀注意、故障红危险动作。
  - 设计风格：工业实用、硬边、仪表化、状态透明，信息密度中等偏高但不做娱乐化视觉。
  - 视觉效果：削角面板、信号轨、扫描线、噪点纹理、数据读数和轻量进入动效。
- 本项目继承：WebTorrent Desktop 的“添加来源后立即尝试播放”、视频监控工作台的“关键指标一眼可见”、Mux Player 的“播放器区域优先”。
- 本项目不继承：Mux 品牌资产和营销页结构、WebTorrent Desktop 的完整桌面 BT 能力、Mux Player 的托管视频平台能力、影视资源站首页、海报墙、白底紫色渐变、廉价科技大屏、过度霓虹和装饰性光效。
- 适配理由：产品核心不是发现内容，而是验证用户提供来源能否播放；主视觉需要让用户感觉自己在操作一个可信的本地信号甲板，播放区像监视器一样聚焦当前媒体，状态区像仪表一样明确说明 WebRTC、Service Worker、peers 和 stream server 的边界。

## 设计定位

- 设计母题：黑匣子信号甲板。
- 核心记忆点：中央播放监视器被信号轴、扫描线和四角准星包围；左右两侧分别承担来源注入和诊断仪表。
- 基调选择：工业实用风，混入少量复古未来主义仪表感。
- 服务对象：技术用户、个人合法内容使用者、需要理解 WebTorrent 播放状态的调试用户。
- 使用场景：桌面浏览器中粘贴 magnet 并观察播放状态；移动浏览器中临时预览任务状态和播放。

界面应让用户第一眼知道三件事：

- 当前有没有可播放任务。
- 播放为什么卡住或失败。
- 下一步应该输入来源、选择文件、等待 peers、切换文件还是停止任务。

## 已确认方向与取舍

- 已确认方向：完全参照“黑匣子信号甲板”概念图的视觉系统执行，但不复制概念图中不属于当前产品能力的假数据、假按钮或假指标。
- 可照搬的部分：顶部三段式能力状态甲板、品牌旁合法来源标记、左侧来源注入舱、文件表格轨、中央播放监视器、任务信息 04、任务操作 05、右侧状态/能力/事件/诊断纵列、硬边面板、扫描线、噪点纹理和数据读数。
- 需要替换的部分：概念图里的示例影视文件、固定时间、NAT/DHT 等当前浏览器 V1 不真实承诺的指标，以及暂停/移除等当前状态机未实现的操作。
- 取舍理由：视觉应尽量接近概念图；行为和数据必须来自真实 WebTorrent runtime，不能为了视觉相似制造不可交互或误导用户的界面。

本项目的界面应：

- 保持播放区优先，状态和文件列表围绕播放决策服务。
- 使用紧凑但不拥挤的信息密度，避免把每个指标做成独立大卡片。
- 失败状态用普通语言解释，并提供下一步动作。
- 让合法使用提醒自然出现，不做吓人的阻断式大弹窗，除非用户试图使用明显无效输入。

不应呈现：

- 资源网站、影视海报站或“热门内容”产品心智。
- 满屏蓝紫渐变、发光边框、粒子背景和装饰性大屏图表。
- 嵌套卡片、按钮层级混乱、无意义图标、文字被播放器遮挡。

## 设计优先级

- P0：核心流程可完成，内容可读，交互可理解，无关键文字截断、遮挡或重叠。
- P1：信息架构清楚，播放器、输入、文件列表和状态面板层级稳定，响应式可靠。
- P2：暗色工作台气质统一，排版节奏舒适，动效服务状态变化。
- P3：可替换素材、细节图标和非核心微交互。

当视觉效果与 P0/P1 冲突时，先保证可读、可用和主流程，再调整视觉表现。

## 目标平台与视口

- 主要平台：现代桌面浏览器，兼容移动浏览器的核心流程。
- 关键视口：桌面 `1440x900`、笔记本 `1280x800`、移动 `390x844`、窄屏 `360x740`。
- 输入方式：键盘、鼠标、触控、文件拖放或文件选择。
- 深浅色模式：V1 暗色优先；如果实现浅色，必须通过同一 token 系统派生。

## 信息架构

主要区域：

- 顶部状态甲板：产品名、合法来源标记、WebRTC / Service Worker / Stream server 三格能力状态、本地优先提示。
- 合法使用提示条：说明仅播放用户自有或合法授权内容，并明确浏览器版 WebTorrent 的 WebRTC/Web Seed 边界。
- 来源注入舱：magnet textarea、`.torrent` 文件导入、开始/清空按钮、输入校验。
- 文件轨：表格式文件列表，占位态显示等待可播放文件，metadata ready 后展示文件名、大小、类型、进度和选择操作。
- 播放监视器：视频或音频播放器、空态准星、信号轴、缓冲状态、当前文件名、播放错误。
- 任务信息 / 操作：当前文件、播放时间、任务状态、停止任务、展开诊断。
- 状态 / 能力 / 事件 / 诊断纵列：进度、peers、速度、ratio、stream server、失败原因、恢复动作和技术细节。

桌面布局：

- 默认三段：左侧来源与文件列表，中间播放器，右侧状态/诊断。
- 播放器宽度优先，状态面板不可挤压播放器到不可用。

移动布局：

- 单列：顶部能力摘要、合法提醒、来源输入、文件列表、播放器、停止/诊断操作、状态与事件。
- 文件列表默认占位展示；metadata ready 后展开文件项，长文件名和长 magnet 必须在列内截断或换行，不产生横向滚动。

## 核心页面 / 屏幕

### 主工作台

- 目标：完成输入来源、解析 torrent、选择媒体和播放。
- 主要内容：来源输入、播放区、任务状态、文件列表、错误恢复。
- 主要操作：添加来源、选择 `.torrent`、选择文件、播放/暂停、停止任务、展开诊断。
- 状态覆盖：空、输入校验失败、metadata 加载、无 peers、无可播放文件、Service Worker 失败、播放错误、播放中、已停止。
- 响应式要求：桌面分栏，移动单列；播放器、输入按钮和错误文案不得重叠。

### 诊断详情

- 目标：让技术用户理解失败发生在 WebRTC、metadata、peers、Service Worker、codec 还是 WebTorrent error。
- 主要内容：错误代码、用户可读原因、技术细节、建议动作、最近事件。
- 主要操作：展开诊断、查看错误代码与技术细节、停止任务、清理最近来源。
- 状态覆盖：无错误、有警告、致命错误、恢复中。
- 响应式要求：桌面可为侧栏或抽屉，移动为折叠区；不遮挡播放器核心控制。

## 设计系统

### 色彩

- 背景：碳黑 `#070806` 和深黑 `#020302` 作为应用与播放器底色。
- 面板：枪灰/橄榄黑层级，用边框、内线和削角区分，不使用通用卡片阴影堆叠。
- 文本：骨白作为主文本，灰橄榄作为辅助文本，避免大面积纯白。
- 主强调色：酸性黄绿用于开始解析、品牌记忆点和关键操作。
- 信号色：冷青用于 peers、下载、上传和连接信号。
- 注意色：琥珀用于合法提醒、等待、metadata、缓冲和 no-peers。
- 危险色：故障红只用于停止任务、错误和破坏性动作。
- 背景深度：不使用彩色渐变；使用噪点纹理、扫描线、硬线网格、信号轨和层叠透明度。

### 字体与排版

- 标题 / 控件字体：`Chakra Petch`，用于品牌、面板标题、按钮和仪表标签。
- 正文字体：`IBM Plex Sans`，用于中文说明、错误恢复和普通状态语言。
- 数字 / 代码字体：`JetBrains Mono`，用于速度、ratio、infoHash、时间、文件大小和诊断代码。
- 中文字体：通过系统中文字体 fallback 渲染，不额外打包中文字体。
- 层级规则：页面品牌强，面板标题小而硬，指标以紧凑数据行和进度轨表达，不做营销式超大数字。
- 行高与字距：默认字距为 0；不使用负字距。

### 间距与布局

- 页面最大宽度：桌面允许全宽工作台，但核心内容有最小宽度约束。
- 栅格 / 分栏：桌面 `280-360px / minmax(520px, 1fr) / 280-340px`，视口不足时右栏下移或折叠。
- 常用间距：8px 基础单位，区域间距 16-24px。
- 圆角：基本不用大圆角，面板采用 2-6px 小半径和 `clip-path` 削角。
- 边框 / 阴影 / 材质：硬边边框、内线、噪点、扫描线和低对比投影；不使用玻璃拟态或发光堆叠。

### 组件规则

- 按钮：主要按钮用于开始/播放；次要按钮用于导入、重试、展开；危险按钮用于停止/清理。
- 输入控件：magnet 输入应支持长文本换行；错误提示贴近输入区。
- 导航：V1 不做多页导航；用工作台区域和折叠面板组织。
- 卡片 / 面板：面板用于功能区域，不嵌套卡片；指标以列表或紧凑行展示。
- 图标：只用于增强识别，必须有文本或 tooltip；不要用图标替代不熟悉操作。
- 表格 / 列表：文件列表优先显示文件名、大小、类型、进度和操作；长路径可中间截断并保留 tooltip。
- 参考风格适配：以工作台密度和播放器沉浸感为准，不做娱乐化海报视觉。

## 动效与交互

- 页面进入：顶部、来源、文件、播放器、状态按顺序轻量 stagger reveal。
- 导航切换：面板展开收起不超过 180ms，尊重减少动态效果偏好。
- 列表 / 卡片出现：metadata ready 后文件列表轻量出现，不逐项大幅动画。
- 监视器动效：播放器空态使用 CSS 扫描线和信号轴；不引入 canvas 或复杂图表。
- hover / focus / active：所有可点击控件必须有清楚状态，focus ring 可见。
- 减少动态效果偏好：关闭非必要动效，只保留状态变化。

## 可访问性

- 文本对比度：正文与背景达到可读对比；错误和提示不只依赖颜色。
- 键盘导航：输入、导入、开始、文件选择、播放器控制、停止和诊断展开可键盘访问。
- 焦点状态：focus ring 不被裁切，不被播放器遮挡。
- 触控尺寸：移动端主要按钮和列表操作高度不低于 44px。
- 屏幕阅读器语义：播放器状态、错误、进度和文件选择使用语义标签或 live region。
- 动效降级：遵守 `prefers-reduced-motion`。

## 视觉验收

完成前应检查：

- 桌面 `1440x900`：空状态、metadata 加载、播放中、错误状态。
- 笔记本 `1280x800`：播放器、左右面板、长 magnet 输入和长文件名。
- 移动 `390x844` 与 `360x740`：单列布局、文件列表、错误文案、主要按钮、长 magnet 和长文件名。
- 关键交互：输入非法 magnet、导入 `.torrent`、metadata ready 后选择文件、无 peers、停止任务。

验收标准：

- 主要页面在目标视口下无重叠、无破碎、无关键文本截断。
- 主要交互状态清楚，包括 hover、focus、active、disabled、loading、empty、error。
- 视觉气质与 `docs/PRODUCT.md` 的产品定位一致。
- UI 技术实现与 `docs/ARCHITECTURE.md` 的工程边界一致。
